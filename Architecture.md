# Architecture.md — 日间/夜间主题模块架构说明

## 项目概述

DayNightModule 是一个纯前端的日间/夜间主题切换模块，提供自动（基于时间）和手动两种主题切换方式，支持用户偏好持久化。

## 架构图

```
┌─────────────────────────────────────────────┐
│                浏览器页面                      │
│  ┌─────────────┐    ┌──────────────────┐    │
│  │  theme.css   │    │ theme-manager.js │    │
│  │  (样式层)    │◄───│   (逻辑层)       │    │
│  └──────┬──────┘    └────────┬─────────┘    │
│         │                    │               │
│    CSS 变量切换          JS 状态管理          │
│         │                    │               │
│    ┌────▼────┐    ┌─────────▼──────────┐    │
│    │ 日间主题 │    │  localStorage      │    │
│    │ 夜间主题 │    │  (偏好持久化)       │    │
│    └─────────┘    └────────────────────┘    │
└─────────────────────────────────────────────┘
```

## 模块职责

### theme.css（样式层）

- 定义日间/夜间两套 CSS 自定义属性（变量）
- 通过 `[data-theme="night"]` 选择器切换主题
- 仅提供主题切换按钮样式（`.dnm-toggle`），不定义通用组件样式
- 尊重 `prefers-reduced-motion` 减少动画设置
- 按钮支持 `:focus-visible` 无障碍焦点样式

### theme-manager.js（逻辑层）

- 基于当前时间自动判断应使用日间还是夜间主题（支持跨午夜区间）
- 提供手动切换主题的 API
- 使用 `localStorage` 保存和恢复用户偏好（带 try-catch 容错）
- 自动在页面挂载切换按钮，BEM 命名避免冲突
- 通过自定义事件通知外部主题变化
- `destroy()` 完整清理定时器和事件监听器

## 数据流

```
页面加载
  │
  ▼
ThemeManager 初始化
  │
  ├──► 读取 localStorage（用户偏好）
  │      │
  │      ├── 有保存值 ──► 使用保存的主题
  │      └── 无保存值 ──► 根据当前时间自动判断
  │
  ▼
设置 data-theme 属性 ──► CSS 变量生效 ──► 页面主题切换
  │
  ▼
派发 theme:initialized 事件
```

用户手动切换时：

```
用户点击切换按钮
  │
  ▼
toggleTheme() / setTheme()
  │
  ├──► 更新 data-theme 属性
  ├──► 保存到 localStorage
  └──► 派发 theme:changed 事件
```

## 关键技术点

1. **CSS 自定义属性**：通过 `:root` 和 `[data-theme="night"]` 两套变量实现主题切换，无需切换样式表
2. **`data-theme` 属性**：作为 CSS 选择器的锚点，同时作为状态标识
3. **`localStorage` 持久化**：用户偏好跨会话保留
4. **自定义事件**：`theme:initialized`、`theme:themeChanged`、`theme:destroyed`，便于外部监听
5. **`data-*` 配置属性**：通过 HTML 属性配置时间区间、存储键名等，零配置开箱即用
